List of rasterisable and raytraceable primitives

This is an incomplete list of possible primitives in computer graphics, as well as algorithms that go with them and links to descriptions of the algorithms if available.

Primitives for rasterisation

Common for Rasterisaton

These primitives are common — especailly in 3D graphics — because they are easy to work with.

  • Points
  • Lines
    • Naïve (solve \(y = rx + c\)) [Wikipedia]
    • DDA [Wikipedia]
    • Midpoint line generation algorithm [GeeksforGeeks]
    • Bresenham's line algorithm [Wikipedia]
    • Xiaolin Wu's line algorithm (with AA) [Wikipedia]
    • Gupta-Sproull algorithm (with AA)
    • Signed distance function
  • Triangles
    • Edge function [Scratchapixel]
    • Barycentric coordinates [Scratchapixel]
    • Scanline rasterisation [Wikipedia]
    • Signed distance function
  • Quadrilaterals
    • Edge function extended

Rasterisable

Other primitives for which at least one algorithm exists for rasterising explicitly - that is, it doesn't require tesselation and then using some other algorithm. Not every one of these will be easy to use in 3D.

  • Rectangles
    • Naïve
  • Elipses and circles
    • Midpoint circle algorithm [GeeksforGeeks]
    • Signed distance function
  • Bezier curves
    • Naïve (find the y-values for given x-value)
    • Bresenham extended [Alois Zingl]
    • Signed distance function
  • Beziergon and SVG paths
    • Loop-Blinn 2005 [GPU Gems | paper]
    • Wavelet rasterisation [paper]
  • Convex polygons
    • Edge function extended
  • Splines

Tesslatable

These are primitives for which I am not aware of any explicit algorithm for when rasterising.

  • Concave polygons

Primitives for ray tracing

Analytical

Shapes for which ray intersection locations can be found analytically.

  • Points
  • Lines
  • Triangles
  • Planes
  • Quadratic surfaces
    • Spheres
  • Cubic surfaces
  • Quartic surfaces

Not technically related, but worth mentioning:

  • Constructive solid geometry

Intersectable

Shapes for which ray intersections can be found, but might require numerical methods. This is often since their roots cannot be found analytically using reasonable methods.

  • Bezier surfaces
  • B-spline and NURBS surfaces
  • n-degree single-variable polynomial surfaces
